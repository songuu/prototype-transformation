# 阶段3：Figma 挂载

## 目标

在初始文档每个页面/弹窗章节顶部插入 Figma 链接，让开发人员看文档时能一键跳转设计稿。

> 纯机械挂载，**主 CLI 直接执行**，不需要 Opus，不需要 Codex。

---

## 输入

| 项目 | 说明 |
|------|------|
| 初始文档 | 阶段2人工确认后的版本 |
| Figma 链接清单 | 任意格式，Claude 自动解析 |

**链接清单支持任意格式：**
```
列表页：https://www.figma.com/design/xxx?node-id=104-122
创建弹窗 https://figma.com/...
编辑弹窗的链接是这个 https://figma.com/...
```

---

## 执行步骤

### 1. 解析清单，识别缺口

将链接整理为映射表，与初始文档页面/弹窗列表对比。

若有未提供链接的页面，**询问用户**：
```
⚠️ 以下页面/弹窗未提供 Figma 链接：
- 删除确认弹窗
请提供链接，或回复「标注为待设计」跳过。
```

---

### 2. 执行挂载

**页面章节顶部：**
```markdown
## 知识库列表页

> 🎨 **设计稿：** [Figma →](https://www.figma.com/design/xxx?node-id=104-122&m=dev)
```

**同一页面多状态时分别挂载：**
```markdown
> 🎨 **设计稿（创建态）：** [Figma →](链接1)  
> 🎨 **设计稿（编辑态）：** [Figma →](链接2)
```

**未提供链接：**
```markdown
> 🎨 **设计稿：** ⏳ 待补充
```

---

### 3. 文档末尾追加索引

```markdown
---
## 附录：Figma 设计稿索引

| 页面/组件 | 状态 | 链接 |
|---------|------|------|
| 知识库列表页 | ✅ 已挂载 | [查看 →](链接) |
| 删除确认弹窗 | ⏳ 待补充 | — |

已挂载 N 个 ／ 待补充 N 个
```

---

### 4. 确认提示

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📄 阶段3 · Figma挂载  完成
产物：requirements-v2-中间文档.md

✅ 已挂载：N 个  ／  ⏳ 待补充：N 个

✅ 确认无误 → 回复「确认」进入阶段4
🔗 有链接需要更正/补充 → 直接提供
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## 注意

- Figma 链接使用 `?m=dev` 参数（开发模式）
- **不修改**初始文档的任何功能描述内容，只做链接插入

## 反模式

- ❌ 链接不匹配时不询问，直接跳过
- ❌ 顺手修改了功能描述内容
- ❌ 产出后自动进入阶段4
